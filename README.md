# Swimming Card Management System

[![Language](https://img.shields.io/badge/language-C%2B%2B-blue)](https://isocpp.org/)
[![Platform](https://img.shields.io/badge/platform-Windows-lightgrey)]()

基于 C++ 的游泳卡管理程序，用于管理游泳卡（校园卡/教师卡）的充值、消费和账单记录。

小组成员：苏小舟、邵凯、欧宇航

---

## 项目简介

**Swimming Card Management System** 校园游泳卡管理系统，支持**学生卡**和**教师卡**两种卡片类型。系统提供充值、消费（扣费游泳）、账单查询等核心功能，并通过二进制文件实现数据持久化。

学生卡每学年享有 **20 次免费游泳** 额度（每年⑨月重置），超出后每次消费5元；教师卡无免费额度，每次消费10元。

---

## 功能特性

| 功能 | 说明 |
|------|------|
| **创建学生卡** | 输入学号（11 位数字）、姓名、性别、所属单位 |
| **创建教师卡** | 输入工号（7 位数字）、姓名、性别、所属单位 |
| **卡登录** | 通过卡号登录后执行操作 |
| **充值** | 为卡充值（1~99999 元） |
| **消费（游泳）** | 扣除对应费用并记录账单 |
| **账单查询** | 查看所有历史交易记录 |
| **数据持久化** | 程序退出时自动保存到 `cards.dat` |
| **输入校验** | 卡号格式验证、余额检查、日期合法性检查等 |

## 核心类说明

| 类 | 描述 |
|----|------|
| `Bill` | 账单记录，包含卡号、交易金额、交易后余额、时间戳 |
| `Card` | 卡片基类，含基本信息、余额管理、账单记录、文件序列化 |
| `StudentCard` | 学生卡，继承自 `Card`，每年 20 次免费额度，每次消费 5 元 |
| `TeacherCard` | 教师卡，继承自 `Card`，无免费额度，每次消费 10 元 |
| `CardManager` | 卡片管理器，维护所有卡片的哈希表，负责文件读写 |

## 快速开始

### 环境要求

- Windows 操作系统（依赖 `<conio.h>`）
- C++17 或更高版本编译器
- CMake（可选）

### 编译与运行

#### 使用 CMake

```bash
mkdir build && cd build
cmake ..
cmake --build .
./Swimcaco3
```

#### 使用 g++ (MinGW)

```bash
g++ -std=c++17 -o Swimcaco3.exe *.cpp
./Swimcaco3.exe
```

## 使用指南

### 主菜单

```
--------------------------------
 Swimming Card Manager
--------------------------------
 1. Create Student Card     创建学生卡
 2. Create Teacher Card     创建教师卡
 3. Login                   登录
 4. About                   关于
 0. Exit                    退出
--------------------------------
```

### 创建学生卡

1. 选择 `1` 进入创建学生卡
2. 输入 **学号**（需为 **11 位数字**）
3. 输入 **姓名**
4. 选择 **性别**（0: 男, 1: 女）
5. 输入 **所属单位**（如学院名称）
6. 创建成功后自动返回主菜单

### 创建教师卡

1. 选择 `2` 进入创建教师卡
2. 输入 **工号**（需为 **7 位数字**）
3. 输入 **姓名**
4. 选择 **性别**（0: 男, 1: 女）
5. 输入 **所属单位**（如学院名称）
6. 创建成功后自动返回主菜单

### 卡操作菜单

登录成功后进入操作菜单：

```
--------------------------------
 Card Operations
--------------------------------
 Balance: 100                 当前余额
 1. Charge                    充值
 2. Consume (swim)           消费（游泳）
 3. View Bills               查看账单
 0. Logout                   退出登录
--------------------------------
```

- **充值**：输入金额（1~99999），余额增加
- **消费**：学生卡每次扣 5 元（优先使用免费次数），教师卡每次扣 10 元
- **查看账单**：显示所有历史交易记录（时间、金额、余额）

### 消费规则对比

| 卡片类型 | 免费额度 | 每次消费金额 | 卡号格式 |
|---------|---------|-------------|---------|
| 学生卡 | 每年 20 次（9 月重置） | 5 元 | 11 位数字 |
| 教师卡 | 无 | 10 元 | 7 位数字 |

## 数据持久化

系统使用 **二进制文件** `cards.dat` 进行数据持久化：

- **自动加载**：程序启动时自动读取 `cards.dat`
- **自动保存**：选择 `退出` 时自动保存所有数据
- **序列化格式**：使用 `operator<<` / `operator>>` 进行二进制读写，包含字符串长度编码和原始数据写入
- 支持卡片信息、挂失状态、余额及所有账单记录的完整持久化

## 技术细节

- **语言标准**：C++17
- **输入处理**：使用 `<conio.h>` 的 `getch()` 实现逐字符输入，支持退格键
- **时间处理**：使用 `<ctime>` 的 `time_t` 和 `tm` 结构体
- **数据存储**：`std::unordered_map` 管理卡片集合，`std::vector` 管理账单记录
- **多态应用**：通过虚函数 `consume()` 和 `isProperId()` 实现学生卡与教师卡的差异化行为
- **文件格式**：自定义二进制序列化（`writeStr`/`readStr` 处理 `std::string`）

### 学生卡学年重置逻辑

```
每年 9 月 1 日后首次消费时，免费次数重置为 20 次
逻辑：比较上次消费日期与当前日期所属的"学年"（9 月~次年 8 月）
```
